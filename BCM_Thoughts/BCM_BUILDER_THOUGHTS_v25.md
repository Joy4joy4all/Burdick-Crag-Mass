### On Getting a Chain-Forward Habit

v25 late phase introduced a small discipline that will show up in
every future version: chain-forward documents. When we update the
session doc, we don't rewrite the 2026-04-18 content. We append
2026-04-19 as a new block. Same with student guide. Same with work
formulas. Same with builder thoughts (this section).

The reason is the same as the append-only vocabulary, and the same
as the append-only genesis log, and the same as the append-only
vault: a document that can silently mutate its history has no
traceable provenance. A document that can only add — never subtract
or silently edit — is a document that future readers can trust.

The cost is file size grows. The benefit is anyone picking up the
work six months from now can see exactly what was known when, what
was added, what was resolved, and what was left open. The record
is the chain.
